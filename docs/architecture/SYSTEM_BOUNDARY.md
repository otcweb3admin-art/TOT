# SYSTEM_BOUNDARY.md — 系统边界

## 当前系统定位

> 当前系统第一阶段是**内部工作流工具**，服务于内部小团队，帮助完成海外华人实体商家的线上增长交付。

---

## 系统是什么

| 是什么 | 说明 |
|--------|------|
| 内部工作流工具 | 支持团队内部标准化流程执行 |
| 商家交付管理系统 | 管理从接入到复盘的完整交付周期 |
| AI 辅助 + 人工审核协作平台 | AI 生成建议，人工决策推进 |
| 模板驱动的交付引擎 | 所有产物基于标准模板生成 |

---

## 系统不是什么

| 不是什么 | 说明 |
|----------|------|
| 完整 SaaS 平台 | 第一阶段无多租户、无自助注册、无公开售卖 |
| 外包市场 | 不对接第三方执行者市场 |
| 自动营销平台 | 不自动执行营销动作，只生成方案 |
| 社交媒体自动发布系统 | 不对接任何社交媒体平台 API 自动发布内容 |
| 完整移动端 App | 第一阶段以 Web 内部工具为主 |
| 全自动 AI Agent | 所有 AI 产物必须经人工审核，不存在无监督自动闭环 |

---

## 系统边界示意

```
[外部]                  [系统边界]                    [内部]
商家提供信息  →→→  TB-001 采集表单  →→→  内部工作流
                                          ↓
                                     AI 生成报告
                                          ↓
                                     人工审核确认
                                          ↓
                                     状态流转推进
                                          ↓
                                     任务分派执行
                                          ↓
                                     数据复盘输出
```

---

## 扩展边界（未来阶段，当前禁止实现）

以下功能仅作为远期规划记录，**当前阶段禁止实现**：

- 多租户 SaaS
- 商家自助入驻
- 外包执行者市场
- 社交媒体平台对接
- 移动端 App
- 自动化营销执行
