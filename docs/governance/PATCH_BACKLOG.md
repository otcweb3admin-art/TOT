# PATCH_BACKLOG.md — 系统补丁池

---

## 文档说明

### 什么是 Patch

Patch 是在推进模板链路过程中**已发现、已确认需要修复或补充，但暂时不打断当前阶段**的优化项。

Patch 不是未来想法，不是产品需求，不是功能扩展。

### 什么时候进入 Patch

满足以下任意一条时，记录为 Patch：

- 推进某一模板时发现上游模板存在字段缺失
- 发现某个输入/输出环节的逻辑漏洞
- 发现模板链路中某处与实际业务不符
- 发现已有内容与治理规则存在冲突
- 发现遗漏的关键字段影响下游输出准确性

### 什么不能进入 Patch

- 未来想做的新功能（进需求池）
- 对系统定位的调整（进 PROJECT_PLAN.md）
- 与当前链路无关的想法
- 模糊的「感觉可以优化」（需先确认再记录）

### 核心原则

```
先记录
不插队
不打断当前阶段
先闭环主链路
后统一 Patch 修复
```

---

## Patch 状态定义

| 状态 | 说明 |
|------|------|
| `recorded` | 已记录，待确认是否真实需要修复 |
| `approved` | 已确认需要修复，等待排期 |
| `scheduled` | 已排入修复计划，有明确修复阶段 |
| `patched` | 已完成修复，关闭 |
| `rejected` | 经评估不采纳，说明原因后关闭 |

## Patch 记录格式

```
Patch ID:      P-XXX
标题:          [简短描述问题]

发现阶段:      [在推进哪个模板时发现]
涉及模板:      [哪些模板受影响]
影响范围:      [对哪些输出/判断产生影响]

问题描述:      [具体缺少什么/哪里不对]

为什么现在不改: [不打断当前阶段的原因]
计划修复阶段:  [第几轮 Patch Round / 条件成熟后]

状态:          recorded / approved / scheduled / patched / rejected
```

### P-001 投流意愿与预算字段缺失

```
Patch ID:      P-001
标题:          TB-001 缺少投流意愿、投流预算与 ROI 预期字段

发现阶段:      TB-005 直播规划模板链路推进过程中发现

涉及模板:
  - TB-001 模块6 增长目标
  - TB-006 引流转化模板（下游依赖方）

影响范围:
  - 广告投放路线判断
  - 引流转化方案中的付费渠道建议
  - 时间预期与增长速度匹配
  - 预算与目标的合理性校验
  - ROI 预期管理

问题描述:
  当前 TB-001 模块6（增长目标）缺少以下字段：
    1. 是否有投流意愿（接受/可尝试/不接受）
    2. 可用于投放广告的月预算区间
    3. 是否接受测试期（前期效果不稳定的适应期）
    4. 可接受的最高获客成本
    5. ROI 预期（期望每花 $1 广告费带来多少回报）
  
  上述字段缺失将导致 TB-006 引流转化模板在制定付费引流方案时
  无法准确判断商家的广告边界，存在方案落地后被商家拒绝的风险。

为什么现在不改:
  当前阶段目标为打通 TB-001 至 TB-008 完整模板链路。
  修改 TB-001 已审核通过的字段结构需要走变更流程，
  会打断主链路推进节奏，因此先记录，待链路闭环后统一处理。

计划修复阶段:  第一轮 Patch Round（主链路闭环后执行）

状态:          patched

修复说明:
  修复时间：Patch Round 1
  修改文件：
    - TB-001-MERCHANT-DIAGNOSIS.md（模块6新增「投流意愿与预算」字段组，共6个字段）
    - TB-006-TRAFFIC-CONVERSION.md（Output-6 升级为四步投流建议结构）
    - TB-007-DATA-REVIEW.md（Input-5 补充5项投流数据字段，Output-5 投流问题行细化）
    - TB-008-90DAY-GROWTH.md（Output-5 投流路线落地，Output-6 投流风险量化，Output-9 行动方案具体化）
  修复原则：兼容追加，不改变原有字段结构和模板顺序
  影响范围：涉及投流决策的下游输出已同步更新，主流程链路不受影响
```
