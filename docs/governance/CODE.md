# CODE.md — 项目最高规则

> 本文件是所有参与者（人工 + AI）必须遵守的最高准则。
> 任何行动与本文件冲突时，以本文件为准。

---

## 禁止事项

| 禁止行为 | 说明 |
|----------|------|
| 禁止未经确认写业务代码 | 所有业务功能必须先经需求确认，再进入开发 |
| 禁止提前平台化 | 不得提前引入多租户、开放 API 等平台化设计 |
| 禁止大规模重构 | 不得在未明确目标的前提下重构现有结构 |
| 禁止擅自修改目录结构 | 目录变更必须提前说明并确认 |
| 禁止创建未确认模块 | 所有新模块必须在文档中有对应定义后方可创建 |
| 禁止跳过文档直接开发 | 必须先有模板/流程文档，再有代码实现 |

---

## 执行规则

1. **每次任务只能完成一个明确目标**
   - 不得在完成 A 的过程中顺带做 B
   - 不得"顺手优化"

2. **修改前必须说明影响范围**
   - 修改哪个文件？
   - 是否影响其他模块？
   - 是否影响已确认的流程？

3. **修改后必须提交变更摘要**
   - 做了什么
   - 改了什么
   - 当前是否有业务代码改动

---

## 工作流规则

- 文档先行，代码后行
- 模板先行，工作流后行
- 内部工具先行，平台化后行
- 单商家闭环先行，规模化后行
