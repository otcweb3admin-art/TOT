# CHANGE_POLICY.md — 变更规则

## 变更申报要求

任何对文档、模板、流程、代码的变更，必须在执行前说明以下内容：

| 申报项 | 说明 |
|--------|------|
| 变更原因 | 为什么要做这个变更？ |
| 修改文件 | 涉及哪些具体文件？ |
| 是否影响流程 | 是否会改变现有工作流节点或状态流转？ |
| 是否影响模板 | 是否会改变现有模板的输入/输出结构？ |
| 是否影响未来开发 | 是否会对后续开发产生约束或前置依赖？ |
| 是否需要人工确认 | 该变更是否需要人工审核后方可执行？ |

---

## 变更分级

### 低风险变更（可直接执行）
- 文字描述的修正
- 格式调整（不影响结构）
- 新增文档（不修改现有文档）

### 中风险变更（需说明后执行）
- 修改现有文档内容
- 调整模板字段
- 新增工作流节点

### 高风险变更（必须人工确认后执行）
- 修改状态流转模型
- 删除已有文件或节点
- 修改项目核心定位或范围
- 涉及代码结构变更（进入开发阶段后）

---

## 变更记录

每次变更执行完毕后，输出变更摘要，格式如下：

```
变更时间：YYYY-MM-DD
变更文件：[文件路径]
变更内容：[一句话描述]
业务代码改动：无 / 有（说明）
```
