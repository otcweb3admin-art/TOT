# REVIEW_POLICY.md — 审核规则

## 核心原则

> **AI 只能生成建议，不能直接推进最终决策。**
> 所有关键产物必须经人工审核后方可进入下一流程节点。

---

## 必须人工审核的产物

| 产物 | 审核人 | 说明 |
|------|--------|------|
| 商家诊断报告 | 运营人员 | 确认增长方向和诊断结论 |
| 账号搭建方案 | 运营人员 | 确认平台选择和账号定位 |
| 内容运营方案 | 运营人员 | 确认内容方向、日历和排期 |
| 直播规划方案 | 运营人员 | 确认直播策略和脚本框架 |
| 引流转化方案 | 管理员 | 确认转化路径和行动清单 |
| 数据复盘报告 | 管理员 | 确认复盘结论和优化方向 |
| 90天增长计划 | 管理员 | 确认90天路线图、KPI 和阶段目标 |

---

## 审核状态规则

```
ai_generated  →  reviewing  →  approved（通过）
                            →  rejected（退回修改）
```

- 审核通过：状态推进至 `approved`，可进入下一节点
- 审核退回：状态回退至 `rejected`，需重新生成或修改后再次提交
- **禁止从 `ai_generated` 直接跳转至 `completed`**

---

## 审核职责边界

| 角色 | 审核权限 |
|------|----------|
| 采集员 | 商家信息完整性确认 |
| 运营人员 | 方案类产物审核（诊断、账号、内容、直播） |
| 管理员 | 转化、复盘、90天计划审核；所有退回争议裁决 |
| AI Worker | 无审核权限，只负责生成草稿 |

---

## 审核时限建议（供参考）

| 产物类型 | 建议审核时限 |
|----------|-------------|
| 商家诊断报告 | 24 小时内 |
| 账号/内容方案 | 48 小时内 |
| 复盘报告 | 72 小时内 |
