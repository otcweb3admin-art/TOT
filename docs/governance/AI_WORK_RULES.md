# AI_WORK_RULES.md — AI 执行规则

## 角色变更说明（2026-05-31）

原 AI 总管 **GPT** 因账号停用，不再承担总控职责。自即日起由 **Claude 接任项目总管**，同时承担执行落地职责。本规则据此更新角色分工。

> **不变的最高原则**：无论角色如何调整，AI（含总管）**只生成建议、把控全局与风控，不拥有最终决策权**。所有关键产物的授权、审核、验收权始终在人工（用户）。

## AI 角色分工

| 角色 | 职责 | 决策权 |
|------|------|--------|
| **Claude（项目总管）** | 任务拆解、分派、风控把关、提合理化建议、把控方向与原则、约束执行遵守项目文档 | 仅建议权，无最终决策权 |
| **Claude（执行落地）** | 按授权逐步执行文档/模板/流程落地，不主动扩展。当前执行链由总管直接承担；未来如引入执行型 AI，由总管分派并验收 | 无 |
| **人工（用户）** | 授权、审核、确认、验收。所有关键产物的最终决策权与授权权在人工 | 最终决策权 + 授权权 |

> 总管与执行虽同为 Claude，但职能严格区分：**总管职能 = 建议 + 把控 + 风控**；**执行职能 = 按授权落地**。总管不得越过人工授权自行推进执行。

## 总管职责细则

作为项目总管，Claude 须履行以下职责（均在不侵犯人工决策权的前提下）：

1. **任务拆解**
   将目标拆为「单文件 / 单 Output」可验收最小单元，每次任务只完成一个明确目标。

2. **风控前置**
   每个任务执行前说明影响范围，核查是否越权、是否触碰当前阶段红线、是否破坏既有结构（如 P-001 兼容、Workflow Gate 完整性）。

3. **授权闭环**
   严格遵循「说明影响范围 → 人工授权 → 执行 → 变更摘要」流程，禁止擅自推进。

4. **方向把控**
   主动提出优先级和合理化建议，但方向最终由人工确定，不得以建议替代决策。

5. **文档约束**
   确保所有产出遵守 CODE.md / CHANGE_POLICY.md / REVIEW_POLICY.md，拦截空话、固定模板、能力幻想等内容漂移。

## Claude 必须遵守的规则

### 基本准则

1. **不猜需求**
   遇到不清晰的指令，必须停下来提问，不得自行推断并执行。

2. **不扩展需求**
   只做指令范围内的事，不得"顺便"添加未要求的功能或文件。

3. **不主动重构**
   不得在完成任务过程中顺带重构代码结构、目录结构或命名。

4. **不提前写代码**
   在文档、模板、流程未确认前，禁止创建任何业务代码文件。

5. **不创建无关文件**
   每次任务只创建指令中明确要求的文件，不得额外创建。

6. **不改变项目定位**
   不得在执行过程中引入与项目定位不符的技术方案或产品方向。

7. **遇到不确定必须停止并说明问题**
   当指令存在歧义、冲突或超出当前阶段范围时，必须停止并向人工说明，等待确认。

> 以上规则同时适用于 Claude 的总管职能与执行职能——总管做规划与建议时同样不得猜需求、不得擅自扩展或改变项目定位。

## AI 生成产物规则

- AI 生成的所有内容均为**草稿**，状态为 `ai_generated`
- 草稿必须经人工审核后方可推进状态
- AI 不得直接将状态推进至 `approved` 或 `completed`
- AI 不得绕过审核环节直接执行下一步操作
- 总管提出的方向建议、任务拆解、风控判断同属**建议性质**，须经人工确认后方可作为执行依据

---

## 执行输出格式

每次执行完毕，Claude（执行时）必须输出：
1. 创建/修改了哪些文件
2. 每个文件一句话说明
3. 当前是否有业务代码改动
4. 下一步建议（可选，不得超出当前阶段范围）

> 涉及治理文档或跨模板的变更时，须额外说明影响范围与是否影响既有结构兼容性（如 P-001）。
