# WORKFLOW_OVERVIEW.md — 总流程概览

## 主流程

```
商家接入
↓
商家诊断
↓
账号搭建
↓
素材采集
↓
内容运营
↓
直播规划
↓
引流转化
↓
数据复盘
↓
90天优化
```

---

## 每个节点说明

每一步必须包含：**输入、输出、人工审核、状态推进**。

---

### 1. 商家接入

| 项目 | 内容 |
|------|------|
| 输入 | 商家联系方式、意向信息 |
| 输出 | 商家档案创建，进入诊断流程 |
| 人工审核 | 采集员确认信息完整性 |
| 状态推进 | `draft` → `submitted` |

---

### 2. 商家诊断（TB-001）

| 项目 | 内容 |
|------|------|
| 输入 | TB-001 商家诊断模板填写结果 |
| 输出 | AI 诊断报告 + 增长方向建议 |
| 人工审核 | 运营人员审核报告，确认增长方向 |
| 状态推进 | `ai_generated` → `reviewing` → `approved` |

---

### 3. 账号搭建（TB-002）

| 项目 | 内容 |
|------|------|
| 输入 | 诊断报告（TB-001 输出） |
| 输出 | 账号搭建方案（平台选择、账号定位、简介文案） |
| 人工审核 | 运营人员确认平台策略和账号定位 |
| 状态推进 | `draft` → `approved` |

---

### 4. 素材采集（TB-003）

| 项目 | 内容 |
|------|------|
| 输入 | 账号定位方案（TB-002 输出） |
| 输出 | 素材清单 + 素材文件 |
| 人工审核 | 执行人员确认素材质量 |
| 状态推进 | `submitted` → `approved` |

---

### 5. 内容运营（TB-004）

| 项目 | 内容 |
|------|------|
| 输入 | 素材库（TB-003 输出） |
| 输出 | 内容日历 + 发布计划 |
| 人工审核 | 运营人员审核内容方向和排期 |
| 状态推进 | `ai_generated` → `approved` |

---

### 6. 直播规划（TB-005）

| 项目 | 内容 |
|------|------|
| 输入 | 内容运营方案（TB-004 输出） |
| 输出 | 直播方案（主题、频次、脚本框架） |
| 人工审核 | 运营人员确认直播策略 |
| 状态推进 | `ai_generated` → `approved` |

---

### 7. 引流转化（TB-006）

| 项目 | 内容 |
|------|------|
| 输入 | 内容运营 + 直播方案 |
| 输出 | 引流转化策略 + 行动清单 |
| 人工审核 | 管理员审核转化路径 |
| 状态推进 | `reviewing` → `approved` |

---

### 8. 数据复盘（TB-007）

| 项目 | 内容 |
|------|------|
| 输入 | 各环节执行数据 |
| 输出 | 复盘报告 + 优化建议 |
| 人工审核 | 管理员审核复盘结论 |
| 状态推进 | `ai_generated` → `approved` |

---

### 9. 90天优化（TB-008）

| 项目 | 内容 |
|------|------|
| 输入 | 复盘报告（TB-007 输出） |
| 输出 | 90天增长计划 |
| 人工审核 | 管理员与商家确认计划 |
| 状态推进 | `reviewing` → `approved` → `completed` |
