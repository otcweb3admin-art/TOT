## 序言：经验库是 Knowledge 与 Intelligence 之间的桥

```
Knowledge Base（知道什么）
        │  经过：验证 → 沉淀 → 升级 → 抽象
        ▼
Experience Base（被验证有效的增长经验）  ← 本文
        │  经过：切片 → 召回 → 网络化 → 发现规律
        ▼
Growth Intelligence（发现规律、为新商家供能）
```

知识是"世界是什么"，智能是"规律是什么"，而**经验库是把前者炼成后者的那座桥**。没有这座桥，知识只是资料、智能无从涌现。本文定义这座桥的结构、规则与治理。

## 第一章　Experience Definition（经验定义）

### 1.1 四个概念：Case / Experience / Pattern / Intelligence

| 概念 | 中文 | 是什么 | 数量 | 可迁移性 |
|------|------|--------|------|---------|
| **Case** | 案例 | 单次"背景+动作+结果"的真实观察 | 多 | 不可迁移（仅此一例）|
| **Experience** | 经验 | 多个**同场景**案例验证结晶的可复用模式 | 少 | 同场景可迁移 |
| **Pattern** | 模式 | **跨场景/跨行业**抽象出的更普适规律 | 更少 | 跨场景可迁移（须匹配条件）|
| **Intelligence** | 增长智能 | 模式网络 + **发现新规律、为新商家供能**的能力 | 非计数（是能力）| 系统级 |

### 1.2 四者关系：逐级抽象、逐级稀少、逐级值钱

```
Case（一个点·多）──验证+去重+升级──► Experience（一条线·少）
                                          │ 跨场景抽象
                                          ▼
Intelligence（一张网·能力）◄──网络化──── Pattern（一个面·更少）
```

### 1.3 铁律：案例 ≠ 经验

> **单个成功案例，不能直接成为经验。**

一个案例可能是**运气、偶然、特例**。"经验"意味着"在相同条件下反复成立的因果"——它要求**多案例 + 同场景 + 结果一致 + 可复现**（接 Evidence Framework 第 8 章经验准入门槛）。**把单个成功当经验，是经验库最常见、最危险的污染源。**

## 第二章　Experience Hierarchy（经验层级体系 L0–L5）

| 层级 | 名称 | 意义 | 升级靠 |
|------|------|------|--------|
| **L0** | Raw Case（原始案例）| 刚发生、**尚未验证**的"背景+动作+结果"观察 | — |
| **L1** | Validated Case（已验证案例）| 经 MVS 验证、**高置信归因**的真实案例 | MVS 验证 |
| **L2** | Experience（经验）| 多个同场景 L1 结晶的可复用模式 | 多案例+同场景+一致+可复现 |
| **L3** | Industry Experience（行业经验）| 一个行业内跨子场景成立的经验 | 行业内多切片验证一致 |
| **L4** | Cross-Industry Pattern（跨行业模式）| 跨多个行业都成立的抽象模式 | 多个独立行业验证一致 |
| **L5** | Growth Intelligence（增长智能）| 模式网络 + 发现规律、供能新商家的能力 | 跨行业×城市×时间反复验证、成网络 |

### 2.1 与既有 L1–L5 的映射（保持一致）

> growth-intelligence-v1 与 experience-base-v1 使用 L1–L5；本文 V2 细化为 L0–L5。映射如下，**无冲突，仅更精细**：

| V2 (L0–L5) | 既有 (L1–L5) | 变化 |
|-----------|-------------|------|
| L0 Raw Case | （L1 案例的"未验证"部分）| **新增**：把"案例"拆出"未验证"前态 |
| L1 Validated Case | L1 案例的"已验证"部分 | 细化 |
| L2 Experience | L2 经验 | 一致 |
| L3 Industry Experience | L3 行业经验 | 一致 |
| L4 Cross-Industry Pattern | L4 跨行业经验 | 一致（更名"Pattern"突出抽象）|
| L5 Growth Intelligence | L5 增长规律 + 智能网络 | 收敛为"增长智能"能力层 |

> 核心增量：**L0/L1 把"案例"一分为二**——强调"未验证（L0）"绝不能跳过验证直接被当经验。这正是 MVS 作为闸门的位置。

## 第三章　Experience Structure（经验结构 · 八要素）

> **一条经验必须包含以下八项；缺少任何关键项，不得称为经验**（只能停留在案例或假设）。

| # | 要素 | 含义 | 缺失后果 |
|---|------|------|---------|
| 1 | **Context 背景** | 在什么场景（行业×城市×客单×渠道×目标×阶段…）| 无法判断对谁适用 → 误迁移 |
| 2 | **Problem 问题** | 要解决的增长问题（漏斗哪个漏点/能力缺口）| 不知为何而做 |
| 3 | **Action 动作** | 具体做了什么（可执行、非空话）| 无法复用 |
| 4 | **Outcome 结果** | 真实结果（指标变化，E1）| 不知有没有用 |
| 5 | **Evidence 证据** | 支撑的证据级别与来源（E1/E2/E3）| 不可信、不可追溯 |
| 6 | **Confidence 置信度** | 强/中/弱（基于样本/一致性/时效）| 不知有多硬 |
| 7 | **Boundary 适用边界** | 在什么条件下成立、对谁有效 | 被错误外推 |
| 8 | **Failure Risk 失效条件** | 什么时候会失效（平台变/季节变…）| 用过期经验误导 |

### 3.1 为什么八要素缺一不可

- **没有 Context/Boundary** → 经验会被错误迁移（餐饮套牙科）。
- **没有 Evidence/Confidence** → 经验不可信，沦为观点。
- **没有 Outcome（真实结果）** → 这是"方案"不是"经验"。
- **没有 Failure Risk** → 经验会被当成永久真理，过期仍在用。

> **八要素 = 经验的"出生证"。** 一条不带背景、不带证据、不带边界、不带失效条件的"经验"，是危险的伪经验。

## 第四章　Experience Lifecycle（经验生命周期）

```
发现 ─► 记录 ─► 验证 ─► 入库 ─► 升级 ─► 应用 ─► 再验证 ─► 降权/失效
Discover Record Validate Admit Promote Apply  Re-validate Deprecate/Expire
(注意到) (L0存档) (MVS)   (L1)   (L2+)  (复用)  (新结果校验) (时效/被推翻)
   ▲                                    │                        │
   │        新案例确认→强化置信 ─────────┘                        │
   └──────────────────────────────────────────────── 归档保留(永不物理删除)
```

| 阶段 | 动作 | 对应层级 |
|------|------|---------|
| 发现 Discover | 注意到一次"背景+动作+结果" | → L0 |
| 记录 Record | 去标识、打八要素标签 | L0 |
| 验证 Validate | MVS 归因/KPI/ROI 验证 | L0 → L1 |
| 入库 Admit | 高置信者入经验库 | L1 |
| 升级 Promote | 达门槛升 L2/L3/L4 | L1 → L2+ |
| 应用 Apply | 为新商家召回复用 | L2+ |
| 再验证 Re-validate | 用新结果校验是否仍成立 | 全层 |
| 降权/失效 Deprecate | 时效/被推翻 → 降权或退役 | 全层 |

### 4.1 经验不是永久真理，是动态资产

> 经验**被新案例强化**（置信升）、**被时间或矛盾弱化**（置信降）、**会过期**（平台/市场变）。**它是活资产，须持续重估**——这正是第八要素"Failure Risk"和"再验证"环节存在的原因。

## 第五章　Experience Admission Policy（经验准入政策）

> 准入规则**直接引用 Evidence Framework + MVS**，是经验库的第一道、也是最严的闸。

### 5.1 三条准入铁律

```
① 未经 MVS 验证 → 禁止进入经验库
② E0 推测 → 禁止进入经验库（连交付物都不能进）
③ 单案例 / 低置信归因 → 禁止升级为经验（是案例，不是经验）
```

### 5.2 合法准入路径

```
L0 Raw Case
   ▼  MVS 验证（高置信归因，TASK-007）
L1 Validated Case
   ▼  Evidence Framework 准入（E1 真实 + 五要素齐全 + 八要素完整）
入经验库（L1）
   ▼  升级门槛（多案例+同场景+一致+可复现）+ 人工审定
L2 Experience
```

### 5.3 准入引用关系

| 准入检查 | 引用 |
|---------|------|
| 是否经验证 | **MVS**（执行/结果/归因/价值四验证）|
| 证据级别是否达标 | **Evidence Framework**（E0 禁入、E1/E3 准入）|
| 八要素是否完整 | 本文第三章 |
| 是否够升级 | **Growth Intelligence Architecture** 升级门槛 |

> **MVS 是经验库的入口质检员；Evidence Framework 是质检标准。** 二者不通过，L0 永远停在 L0。

## 第六章　Experience Promotion Model（经验升级模型）

```
Raw Case ─► Validated Case ─► Experience ─► Industry Experience ─► Cross-Industry Pattern ─► Growth Intelligence
  L0           L1               L2              L3                       L4                        L5
```

每一级升级**必须满足新的、更高的证据要求**：

| 升级 | 新增证据要求 | 防止 |
|------|-------------|------|
| L0 → L1 | MVS 高置信归因 + 真实结果(E1) | 把未验证当事实 |
| L1 → L2 | 多案例 + 同场景 + 一致 + 可复现 | 单例当规律 |
| L2 → L3 | 行业内多个场景切片验证一致 | 局部当行业 |
| L3 → L4 | 多个**独立行业**各自验证一致 | 行业当普适 |
| L4 → L5 | 跨行业×城市×时间反复验证、成网络 | 把模式误当万能 |

### 6.1 两条升级铁律

1. **禁止跳级升级**：不能把 L0 直接当 L2、不能把 L2 餐饮经验直接当 L4 模式用——**每级都要赚到对应证据**。
2. **人工审定每次升级**：AI（Experience Curator）只能**标候选**，人裁定（接 AIGO 第七章 + Evidence 宪法）。

> **可降级**：高层经验被新证据证伪 → 降级或标注边界（如 L4 被某新行业证伪 → 降回 L3）。升级是赚来的，不是给定的。

## 第七章　Experience Segmentation Model（经验切片模型）

> **经验必须带上下文（切片）。** 切片是经验"被正确迁移"的前提，也是"防止被错误迁移"的护栏。

### 7.1 切片维度（至少九维）

| 维度 | 例子 |
|------|------|
| 行业 | 餐饮/牙科/美业 |
| 国家 | 美国/加拿大/澳洲 |
| 城市 | 洛杉矶华人区 |
| 平台 | TikTok/Maps/小红书 |
| 客单价 | $30 / $3000 |
| 用户群体 | 年轻家庭/银发/留学生 |
| 增长目标 | 拉新/复购/品牌 |
| 增长阶段 | 冷启动/增长/放大 |
| 渠道类型 | 自然/活动/口碑/投流 |

### 7.2 为什么切片是正确迁移的前提

```
❌ 无切片：餐饮经验 = 所有行业经验 → 套到牙科 → 灾难
✅ 带切片：餐饮 × 洛杉矶 × $30 × 高频 × TikTok × 拉新 = 特定场景经验 → 只喂同场景商家
```

> 切片让经验**召回得准、迁移得对**。无切片的经验，要么召回不到（死库存），要么误召回（污染）。**切片是经验从"私有案例"变成"可复用智能"的结构前提。**

## 第八章　Experience Quality Framework（经验质量框架）

### 8.1 质量三级

| 质量级 | 含义 | 可做什么 |
|--------|------|---------|
| **Strong Experience** | 多样、一致、新近、可复现、边界清晰 | 可独立支撑**重决策**（投流/放量）|
| **Medium Experience** | 部分维度不足 | 须佐证后用，作主要参考 |
| **Weak Experience** | 样本少/陈旧/边界模糊 | 仅作提示/候选，**不得单独支撑重决策** |

### 8.2 质量六维

| 维度 | 强 |
|------|----|
| 证据 | E1/E3 硬证据 |
| 样本 | 多且独立 |
| 时效 | 新近 |
| 一致性 | 多案例同向 |
| 可复现性 | 同条件可重复 |
| **边界清晰度** | Context/Boundary/Failure Risk 明确 |

### 8.3 铁律：经验质量 ≠ 案例数量

> - **100 个同源案例（一个商家）< 5 个多样独立案例**：前者过拟合一种风格。
> - **结果矛盾的案例越多越糟**：无一致性。
> - **陈旧案例再多也低质**：时效失效。
>
> **质量是"证据×样本×时效×一致×可复现×边界"的加权，不是案例计数。** 追求"质量加权的经验"，而非"经验数量"。

## 第九章　Experience × Growth Intelligence（经验库 × 增长智能）

### 9.1 分工：保存 vs 发现

| | Experience Base（经验库）| Growth Intelligence（增长智能）|
|---|------------------------|------------------------------|
| 职责 | **保存**经验（L0–L4）| **发现**规律（L5）|
| 形态 | 存储 + 运作机制 | 抽象 + 网络 + 能力 |
| 类比 | 图书馆藏书 | 会读全馆、发现规律的学者 |

### 9.2 智能是经验的抽象层，不是存储层

> **智能不是"存得多"，而是"抽象得出规律、应用得到新商家"。**

```
经验库（存）：L0→L1→L2→L3→L4，一条条带八要素的经验
        │ 增长智能在其上做三件事：
        ▼
增长智能（用）：① 切片召回（给新商家匹配同场景经验）
              ② 跨经验抽象（从多条 L4 模式发现 L5 规律）
              ③ 网络化（经验互相印证/冲突仲裁/组合）
```

> **经验库管"有没有、对不对、过没过期"；增长智能管"在这堆经验里能看出什么规律、怎么用在新商家身上"。** 智能是经验库之上涌现的能力层——库是基础，智能是升华。**存储再大，不抽象、不召回、不成网络，就不是智能**（呼应 V1："1000 条经验 ≠ 增长智能"）。

## 第十章　Competitive Moat（护城河）

### 10.1 核心问题：为什么经验库是 TOT 最核心资产？

因为在"模型平权"的时代，**别的都会平权，唯独经验不会**：

| 资产 | 会平权吗 | 为什么 |
|------|---------|--------|
| AI 模型 | ✅ 会 | 前沿模型对所有人开放，谁都能用同样的 |
| Prompt | ✅ 会 | 可复制、会随模型迭代过时 |
| Agent | ✅ 会 | 框架开源、可抄 |
| **经验库** | ❌ **不会** | 来自真实结果的累积，无法复制、无法购买、无法用时间补齐 |

### 10.2 经验库形成的五重壁垒

```
① 时间壁垒：经验需真实交付累积，1000 家的经验 = 1000 次真实交付的时间，不可压缩
② 数据壁垒：验证数据私有、真实、带结果，竞争者无从获取
③ 验证壁垒：每条经验都过 MVS 验证，假经验进不来——质量本身是壁垒
④ 网络效应壁垒：每个新商家既消费又贡献经验，差距随规模自我放大
⑤ 复利壁垒：成功率↑→口碑↑→更多商家→更多经验→成功率↑（自强化飞轮）
```

### 10.3 为什么经验不会平权

- **模型/Prompt/Agent 是"公共能力"**：谁都能获得，是入场券，不是差异化。
- **经验是"私有的、被验证的、带场景的真实结果"**：它**只能靠真实做、真实验证、真实累积**得来——这是时间 + 数据 + 验证三者的乘积，无法被任何更强的模型替代。

> **最终判断**：在人人都有强模型的时代，**护城河不在"用什么模型"，而在"喂给模型什么专有的、被验证过的、带场景的经验"**。模型会平权，Prompt 会平权，Agent 会平权，**唯独经验库不会平权**——它叠加时间、数据、验证、网络、复利五重壁垒，最终成为 **TOT 最大、且唯一不可复制的护城河**。这就是为什么经验库是 TOT 最核心资产。

## 架构结论（核心要点）

1. **四概念分明**：Case（案例）→ Experience（经验）→ Pattern（模式）→ Intelligence（智能），逐级抽象、稀少、值钱；**案例 ≠ 经验**。
2. **L0–L5 六级层级**：Raw Case → Validated Case → Experience → Industry → Cross-Industry Pattern → Growth Intelligence；细化既有 L1–L5，**L0/L1 拆分强调"未验证不得跳过"**。
3. **经验八要素**：Context/Problem/Action/Outcome/Evidence/Confidence/Boundary/Failure Risk，缺一不得称经验。
4. **经验是动态资产**：发现→记录→验证→入库→升级→应用→再验证→降权/失效，被强化或弱化，非永久真理。
5. **准入引用 MVS + Evidence Framework**：未验证禁入、E0 禁入、单例禁升级。
6. **升级靠逐级赚取的新证据，禁止跳级，人工审定**，可降级。
7. **切片是正确迁移的前提**：至少九维上下文，无切片即死库存或污染源。
8. **质量 ≠ 数量**：Strong/Medium/Weak 决定复用权限；质量是六维加权。
9. **智能是经验的抽象层，不是存储层**：库保存、智能发现规律。
10. **经验库是最核心资产**：模型/Prompt/Agent 会平权，经验不会；五重壁垒（时间/数据/验证/网络/复利）筑成 TOT 最大护城河。
