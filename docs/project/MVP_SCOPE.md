# MVP_SCOPE.md — MVP 范围

## 第一版 MVP 实现范围

| 功能 | 说明 |
|------|------|
| 商家基础信息录入 | 通过 TB-001 模板采集商家信息 |
| 商家诊断模板 | 结构化填写，覆盖基础信息、客户画像、卖点等 |
| AI 生成诊断报告 | 基于采集信息自动生成诊断报告草稿 |
| 人工审核 | 运营人员审核 AI 报告，确认或退回 |
| 审核通过进入下一流程 | 状态推进至「账号搭建」入口 |
| 基础任务状态管理 | 支持 draft / submitted / ai_generated / reviewing / approved / rejected / completed |

---

## 第一版 MVP 暂不实现

| 功能 | 原因 |
|------|------|
| SaaS 多租户 | 第一阶段只服务单一内部团队 |
| 自动结算 | 商业模式尚未锁定 |
| 外包市场 | 超出第一阶段边界 |
| 自动发布社交媒体 | 第三方平台对接复杂，风险高 |
| 完整移动端 App | 内部工具阶段，Web 优先 |
| 全自动 AI Agent | 所有 AI 产物必须经人工审核，不做无人工干预的自动闭环 |

---

## MVP 验收标准

- [ ] 可完整录入一个商家信息
- [ ] AI 可生成该商家诊断报告
- [ ] 运营人员可审核报告并通过
- [ ] 状态正确流转至下一节点
- [ ] 整条流程可无阻跑通一次
