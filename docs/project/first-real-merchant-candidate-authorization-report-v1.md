## 1. 本次任务目标

从候选商家中筛选 1 家最适合进入首轮试运行的真实商家，按授权清单做预评估，输出 **GO / CONDITIONAL GO / NO-GO** 三态结论之一。只有 GO 或明确限定范围的 CONDITIONAL GO，后续才允许进入系统创建真实商家。

## 2. 候选商家筛选标准（十条）

| # | 标准 | 说明 |
|---|------|------|
| 1 | 商家真实存在 | 实体可核实（有店面/有经营），不接受转述中的"朋友的店" |
| 2 | 有明确负责人或联系人 | 能拍板配合事项的具体人 |
| 3 | 愿意沟通线上获客问题 | 对"线上没客流/不会做"有真实痛点 |
| 4 | 有基础产品 / 服务 | 卖点 3 秒可懂（参照 Outreach Kit 画像） |
| 5 | 有一定承接能力 | 客人来了接得住（履约/出餐/响应不崩） |
| 6 | 愿意提供基础资料 | 接受 Field Pack 范围的采集 |
| 7 | 愿意配合客户确认 | 接受"确认通过才进下一步"的节奏 |
| 8 | 接受"不承诺增长结果" | 听完承诺边界话术仍愿意试 |
| 9 | 不要求立即投流 | 接受先补地基再考虑放量 |
| 10 | 适合做小范围试运行 | 规模/复杂度匹配首轮（单店优先，不接连锁多店首单） |
