## 0. 立论：增长不是"做营销"，是"经营系统的系统性改善"

TOT 不是营销建议生成器，不是 AI 工作流系统。它的目标是 **提升真实商家的增长成功率**。

要理解 TOT 如何做到，必须先放弃"营销视角"（涨粉、爆款、曝光），改用**经营视角**：

> **增长 = 让更多人认识你 → 更多人信任你 → 更多人买你 → 更多人复购你 → 更多人推荐你，并让这一切随品牌沉淀而越来越便宜。**

营销指标（粉丝/播放量）是**虚荣指标**；经营增长（客流/咨询/成交/复购/口碑/品牌）才是**真实增长**。TOT 的方法论建立在后者之上。

## 1. Growth Definition（增长定义）

从**商家经营角度**定义六种增长，及其关系：

| 增长 | 经营含义 | 不是 |
|------|---------|------|
| **客流增长** | 更多潜在顾客被触达 / 到店 | ≠ 涨粉、≠ 播放量 |
| **咨询增长** | 更多人主动询问/互动/留资 | ≠ 点赞 |
| **成交增长** | 更多咨询转化为真实购买 | ≠ 加购 |
| **复购增长** | 更多买家重复购买 | ≠ 一次性成交 |
| **口碑增长** | 满意顾客主动推荐、留好评 | ≠ 刷评 |
| **品牌增长** | 累积认知，顾客主动找你、愿溢价 | ≠ logo 好看 |

### 1.1 六者关系：一条漏斗 + 一个复利外环

```
       ┌──────────────── 品牌增长（复利外环：让以下全部变便宜）────────────────┐
       │                                                                       │
       │   客流增长 ──► 咨询增长 ──► 成交增长 ──► 复购增长                       │
       │   （触达）     （兴趣）     （变现）     （留存）                        │
       │                                            │                          │
       │                                            ▼                          │
       └────────────────── 口碑增长 ◄──────── 满意顾客（推荐 + 好评）───────────┘
                              │
                              └──► 反哺客流（更便宜的新客）+ 沉淀品牌
```

**三条关键认知**：
1. **漏斗是乘法不是加法**：客流×咨询率×成交率×复购率，任一环为零则整体为零。"只做客流"是无效增长。
2. **口碑是漏斗末端反哺前端**：满意的成交/复购顾客产生口碑 → 口碑带来**更便宜、更高信任**的新客流（本地实体商家尤其依赖 Google Maps 评价）。
3. **品牌是复利外环**：品牌认知积累后，客流/咨询/成交的**单位成本持续下降**（顾客主动搜索、指名到店、接受溢价）。品牌是慢变量，但决定长期天花板。

> **真实增长 = 整条漏斗改善 + 口碑反哺 + 品牌复利**，而非顶部某个数字变大。这是 TOT 一切方法的出发点。

## 2. Growth Capability Map（增长能力地图）

增长不是"动作"，是**五种能力**的组合。每种能力定义：目标 / 输入 / 输出 / 衡量指标 / 失败原因。

### 2.1 获客能力（Acquisition）
- **目标**：让陌生人认识/到达商家。
- **输入**：内容 + 平台阵地 + 引流路径 + 投流(可选)。
- **输出**：曝光、到店、新客。
- **衡量**：触达量、到店量、新客数、**获客成本**。
- **失败原因**：没内容、押错平台、无引流路径、扎进红海渠道、只曝光不引导。

### 2.2 信任能力（Trust）
- **目标**：把"认识"变成"相信"。
- **输入**：信任素材（口碑/案例/资质）、内容一致性、人设、品牌故事。
- **输出**：关注、咨询意愿、私域加好友。
- **衡量**：关注转化率、咨询率、加好友率。
- **失败原因**：内容空洞假大空、无信任素材、定位漂移、人设与现实不符。

### 2.3 转化能力（Conversion）
- **目标**：把"咨询"变成"成交"。
- **输入**：转化路径、话术、**承接能力**、活动钩子。
- **输出**：成交、客单价。
- **衡量**：咨询→成交率、客单价、转化周期。
- **失败原因**：**无人承接**、Day1 强成交、信任未建就逼单、承接慢导致流失。

### 2.4 复购能力（Retention）
- **目标**：把"买过"变成"再买"。
- **输入**：私域、会员/复购触达、服务体验、复购理由。
- **输出**：复购、顾客终身价值(LTV)。
- **衡量**：复购率、复购周期、私域活跃度。
- **失败原因**：无私域沉淀、低频行业误套高频策略、无复购理由、私域无人维护变僵尸。

### 2.5 品牌能力（Brand）
- **目标**：累积认知，降低全漏斗成本。
- **输入**：一致定位、品牌故事、长期口碑积累、持续内容。
- **输出**：主动搜索、指名到店、溢价能力、自然流量。
- **衡量**：品牌词搜索量、指名率、自然流量占比。
- **失败原因**：急功近利、定位频繁变、无长期积累、为短期增长牺牲定位。

### 2.6 能力 → 漏斗映射

| 能力 | 主要驱动的漏斗环节 |
|------|-------------------|
| 获客能力 | 客流增长 |
| 信任能力 | 客流→咨询 |
| 转化能力 | 咨询→成交 |
| 复购能力 | 成交→复购 |
| 品牌能力 | 口碑+全漏斗成本下降（复利外环）|

## 4. Growth Path Model（增长路径模型）

完整增长路径 + 每阶段阻塞因素 + 对应 TB：

```
陌生人 ─► 关注 ─► 信任 ─► 咨询 ─► 到店 ─► 成交 ─► 复购 ─► 推荐
  │        │       │       │       │       │       │       │
（触达）  （兴趣） （相信） （行动） （线下） （变现） （留存） （裂变）
```

| 阶段跃迁 | 阻塞因素 | 对应 TB | 所需能力 |
|---------|---------|---------|---------|
| 陌生人→关注 | 没内容/没曝光/平台错/内容无吸引力 | TB-002 阵地、TB-004 内容、TB-006 引流 | 获客 |
| 关注→信任 | 内容空洞/无信任素材/定位漂移 | TB-003 信任素材、TB-004 一致内容、TB-005 直播 | 信任 |
| 信任→咨询 | 无引导/无 CTA/无入口 | TB-004 轻引导、TB-006 转化路径+私域入口 | 信任→转化 |
| 咨询→到店 | 无人回复/承接慢/信息不全 | TB-002 Bio/Maps/联系方式、TB-006 即时承接 | 转化(承接) |
| 到店→成交 | 线下体验/价格/无钩子 | TB-006 活动/转化（**线下体验在系统边界外**）| 转化 |
| 成交→复购 | 无私域/无复购理由/低频误套 | TB-006 私域、TB-008 复购路线 | 复购 |
| 复购→推荐 | 无激励/体验平庸/无口碑沉淀 | TB-006 口碑/Maps 评价、TB-008 口碑路线 | 口碑+品牌 |

**两条诚实边界**：
1. **承接是路径的隐形断点**：多数失败不在"引流不够"，而在"咨询/到店后无人承接"（呼应 TB-006"先承接后引流"铁律）。路径设计必须**先保证承接**。
2. **线下环节在系统边界外**：到店→成交的线下体验、产品力、价格，TOT 影响有限。TOT 负责"把对的人高信任地引到店"，**不负责替商家把店开好**——这点要对商家诚实约定。

## 5. Growth Lever Framework（增长杠杆框架）

六个杠杆，每个分析：适用条件 / 风险 / 投入 / 预期效果。

| 杠杆 | 适用条件 | 主要风险 | 投入 | 预期效果 |
|------|---------|---------|------|---------|
| **内容杠杆** | 几乎所有商家（基础杠杆）| 见效慢、需持续产出 | 中（素材+时间）| 长期复利、低成本获客+建信任 |
| **平台杠杆** | 选对匹配客群的平台 | 平台规则变、押错平台 | 低-中 | 放大或限制其他所有杠杆 |
| **口碑杠杆** | 有真实好产品/服务 | 造假违规、负评扩散 | 低 | 高信任、低成本、本地强（Maps）|
| **活动杠杆** | **已有信任基础** | Day1 强促销反噬、损利润 | 中 | 短期拉升，**需承接兜底** |
| **私域杠杆** | 高频或可复购、有人维护 | 无人维护变僵尸、骚扰顾客 | 中（人力）| 复购/LTV、抗平台风险 |
| **投流杠杆 (P-001)** | **有承接 + 有预算 + 接受投流** | ROI 幻想、烧钱无承接、过早投流 | 高（资金）| 快但贵；**放大**已验证路径，不建信任 |

### 5.1 杠杆铁律（贯穿所有模板的增长原则）

1. **杠杆有顺序**：先用**内容/口碑/平台**建基础（建信任、建承接），再用**活动/投流**放大。顺序错 = 烧钱无果。
2. **投流是放大器，不是发动机**：投流只能放大"已验证可承接的路径"，**不能替代信任、不能修复破损漏斗**。无承接就投流 = 把钱倒进漏斗的洞里（P-001 四步：可行性→平台→测试期→ROI 对齐，正是为防此）。
3. **先稳后放**：内容/营业系统稳定优先于增长放量；增长不得挤占内容产能或冲击营业承接。
4. **每个杠杆都要可降级**：高投入杠杆（活动/投流/私域）必须有降级路线，执行受阻时降频而非崩盘。

## 6. Industry Growth Classification（行业增长分类框架）

### 6.1 为什么牙科 ≠ 餐饮

| 维度 | 牙科 | 餐饮 |
|------|------|------|
| 消费频率 | 极低频（年级）| 高频（周级/日级）|
| 客单价 | 高 | 低-中 |
| 决策周期 | 长、慎重 | 短、冲动 |
| 信任门槛 | **极高**（关乎健康/不可逆）| 低-中 |
| 复购逻辑 | 弱（靠推荐）| 强（靠体验+触达）|
| 增长主引擎 | 信任 + 品牌 + 口碑 | 获客 + 复购 + 口碑 |

→ 同一套动作在两个行业效果天差地别：牙科发"9.9 秒杀"会**摧毁信任**，餐饮做"专业科普"则**过重**。**增长逻辑由行业决定。**

### 6.2 四类行业增长框架

| 行业类型 | 代表 | 核心特征 | 能力权重 | 杠杆优先 | 内容方向 | 投流(P-001) |
|---------|------|---------|---------|---------|---------|------------|
| **高信任行业** | 牙科/医美/教育/法律 | 高客单、低频、信任门槛极高 | 信任>品牌>口碑 | 内容/口碑/私域 | 专业/案例/资质/信任 | 谨慎（贵但 LTV 高）|
| **高频行业** | 餐饮/咖啡/美甲 | 低客单、高频、冲动+复购 | 获客>复购>口碑 | 平台/内容/活动/Maps | 场景/食欲/氛围/上新 | 可用（放量到店）|
| **低频高客单** | 婚纱/家装/留学/房产 | 极低频、超高客单、超长决策 | 信任>品牌>口碑 | 内容/口碑/私域长期培育 | 案例/过程/信任 | 谨慎（线索培育）|
| **冲动消费** | 零食/饰品/网红品 | 低客单、冲动、视觉驱动 | 获客>转化 | 内容爆款/平台/投流 | 视觉/爆点/上新 | 激进（放量）|

### 6.3 TOT 如何适配不同行业

**模板不变，参数/权重/经验按行业切片调整**：
- 行业类型决定 ① **能力权重**（先建哪个能力）② **杠杆选择与顺序** ③ **内容方向** ④ **投流策略** ⑤ **KPI 基准**（牙科的"咨询量"和餐饮的"客流量"不可同标）。
- **经验库按行业切片**（呼应知识架构的"行业经验"维度）：牙科的成功模式喂牙科，不串给餐饮。
- TB-001 诊断时即识别行业类型 → 全链按行业框架"配权重"，而非套用统一模板。这是 TOT 规模化后"千行千面"的方法论入口。

## 7. Validation Loop（TB-007 验证中心）

### 7.1 为什么 TB-007 是整个系统的验证中心

TB-007 是**唯一接触"真实结果"的节点**。在它之前，一切都是"方案"（我们以为有效）；只有它把方案放进真实数据里检验，得出"是否真有效"。

> **没有 TB-007，TOT 就是一台开环的营销建议机**——正是 TOT 立项要否定的东西。TB-007 是把系统从"建议机"升级为"增长引擎"的那个闭合动作。

### 7.2 TB-007 验证什么

| 验证对象 | 用什么验证 |
|---------|-----------|
| **能力是否真建立** | 信任能力→咨询率；转化能力→成交率；复购能力→复购率 |
| **杠杆是否真有效** | 各杠杆的真实 ROI / 效果（投流看真实获客成本与 ROI）|
| **路径是否真通** | 漏斗哪个阶段在漏（关注→信任？咨询→成交？）|
| **归因：执行 vs 策略** | 没做到（执行问题，调执行）vs 做到了仍无效（策略问题，要重做）|

### 7.3 方案→执行→结果→经验 闭环

```
方案(TB-002~006)  ──►  执行(真实动作)  ──►  结果(TB-007真实数据)  ──►  经验(去标识结晶)
  我们打算做什么        实际做了什么/花多少      真实客流/转化/ROI         条件→动作→结果模式
                                                      │                        │
                                                      └──── 归因(执行/策略) ────┘
                                                                                │
                                          经验回流 ──► 提升下一轮诊断与方案 ◄────┘
```

TB-007 = 增长闭环的"合上"动作，也是知识架构里 **Experience（经验）** 的唯一生产口。**它既验证当前商家的增长，又为整个系统积累经验。**

## 8. Growth Intelligence Flywheel（经验增长飞轮）

### 8.1 10 家 vs 1000 家：本质区别

| | 10 家客户 | 1000 家客户 |
|---|---------|------------|
| 经验状态 | 个案，无统计 | 多维切片密集、高置信模式 |
| 新商家冷启动 | 靠通用方法论 + 人的水平 + 运气 | 召回"同行业×同城×同渠道"已验证打法 |
| 诊断 | 经验判断 | 数据支撑、有基准对标 |
| 失败 | 事后才知道 | 提前预警（"此条件下做 X 会失败"）|
| 边际成功率 | 持平（靠人）| **递增**（靠经验复利）|
| 本质 | 手工代运营 | **数据驱动的增长智能** |

> 区别不是"量变"，是"质变"：10 家是**人在代运营**，1000 家是**经验在代运营、人在决策**。

### 8.2 经验增长飞轮

```
        更多商家走完闭环
              │
              ▼
   更多带维度标签的真实案例（行业×城市×渠道×内容×投流）
              │
              ▼
   各切片达到统计质量 → 结晶为高置信经验(E3)
              │
              ▼
   新商家按 Context 召回已验证打法 → 增长成功率提升
              │
              ▼
   成功率↑ → 口碑↑ → 更多商家  ↺（复利护城河）
```

**飞轮转动的前提**（来自知识架构铁律）：每个案例**从第一天就打齐维度标签**，否则无法回切片，飞轮空转。**护城河不是模板，是被真实结果验证过、按维度切片的经验库。**

## 9. AI Boundary（AI 在增长体系中的职责）

AI 是**增长证据工作者**，不是增长决策者。四职能：

| 职能 | 在增长语境下做什么 | 不能做 |
|------|-------------------|--------|
| **Researcher 研究员** | 研究行业增长逻辑、竞品打法、平台机制、本地市场 | 把推测当事实 |
| **Analyst 分析师** | 诊断增长瓶颈、找漏斗漏点、归因、对标经验库 | 替商家下结论 |
| **Planner 方案工程师** | 设计能力建设/杠杆组合/路径打通方案（带证据+降级）| 决定走哪条增长策略 |
| **Auditor 审计员** | 审方案是否违反增长铁律（先承接后引流/先信任后转化/先稳后放/P-001）、是否有证据、是否碰红线 | 做最终放行裁定 |

### 9.1 红线

- ❌ **禁止 AI 决定增长策略**（投不投流、放不放量、走哪条路径，均由人工决策）。
- ✅ **AI 提供依据**：诊断、漏斗分析、归因、经验召回、方案草稿、风险预警、置信度。
- ✅ **人类负责决策**：选择路径、批准投放、确认放量、承担结果。

> **一句话**：AI 让增长判断"证据充分、就绪可决"；人决定"往哪长、长多快"。增长铁律是 Auditor 的审计标准——任何违反"先承接/先信任/先稳"的方案，AI 必须拦截并退回。

## 10. Architecture Conclusions（架构结论）

1. **增长是经营漏斗的系统性改善 + 口碑反哺 + 品牌复利**，不是营销虚荣指标。TOT 服务真实增长。
2. **增长 = 五种能力（获客/信任/转化/复购/品牌）的组合**；TB-001~008 是这些能力的**建设工序**，能力由多 TB 协同产生。
3. **增长路径有隐形断点（承接）和系统边界（线下）**：方案须先保承接、对线下边界诚实。
4. **六大杠杆有顺序，投流是放大器不是发动机**：先内容/口碑建基础，后活动/投流放量；无承接不投流（P-001）。
5. **行业决定增长逻辑**：四类行业（高信任/高频/低频高客单/冲动消费）权重与杠杆不同；模板不变，**参数/权重/经验按行业切片**——这是 TOT"千行千面"的入口。
6. **TB-007 是验证中心，闭环靠它合上**：把"以为有效"变成"证明有效"，并生产经验。没有它，系统退回营销建议机。
7. **10→1000 的本质是经验飞轮（边际成功率递增）**：护城河是按维度切片、被真实结果验证的经验库，前提是从第一天打齐维度标签。
8. **AI 提供增长依据，人类决策**：四职能（研究/分析/方案/审计），增长铁律是审计标准，禁止 AI 拍板。

### 10.1 本方法论作为后续工作的上位依据

| 后续工作 | 本方法论给它的约束 |
|---------|-------------------|
| **诊断逻辑设计** | 按"能力基线 + 漏斗漏点 + 行业类型"诊断，而非罗列问题 |
| **方案逻辑设计** | 方案=能力建设+杠杆组合+路径打通，须守增长铁律 |
| **KPI / Metric 设计** | KPI 按行业切片定基准，对齐漏斗各环（非通用默认值）|
| **行业适配设计** | 四类行业框架决定能力权重与杠杆顺序 |
| **经验库 / 飞轮设计** | 案例按 行业×城市×渠道×内容×投流 维度打标签 |
| **AI Agent 设计** | 四职能边界 + 增长铁律审计 + 人工决策 |

> **成功标准自检**：本文不是技术方案，而是一份回答"**商家为什么增长 / 如何增长 / 如何验证增长 / 如何持续增长**"的增长方法论总纲，可指导后续全部增长相关设计。
